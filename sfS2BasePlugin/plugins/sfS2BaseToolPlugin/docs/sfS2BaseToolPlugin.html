動作環境

依存ライブラリは次になります。個別にPEARインストールを実施下さい。

  • symfony-1.0.8
  • PHPUnit-3.1.9
  • s2container.php5-1.2.0
  • s2dao.php5-1.1.2
  • sfS2BasePlugin-0.8.0

ダウンロード

  • 2007-11-18 : sfS2BaseToolPlugin-0.8.0.tgz

インストール

　Symfonyのプラグインとしてインストールします。 

% cd /path/to/symfony/project
% symfony plugin-install sfS2BaseToolPlugin-version.tgz

>> pear      install ok: channel://__uri/sfS2BaseToolPlugin-version
%
% symfony plugin-list

コマンド

s2gendao

コマンドリファレンス

% symfony s2gendao [環境指定]
  • 環境指定
    symfonyの環境を指定します。この値に従ってプロジェクトディレクトリ/config/ディレクトリのpdo.diconが使用されます。 デフォルト値は [prod] です。

説明

　環境指定で指定されたpdo.diconを用いて、全テーブルのdaoとentityを作成します。daoとentityのクラスファイルはプロジェクトディレクトリのlib/ディレクトリに生成されます。


s2at

コマンドリファレンス

% symfony s2at アプリケーション名
  • アプリケーション名
    UnitTestを実行するアプリケーション名。s2atではアプリケーションごとにUnitTestを実行します。

説明

　テストディレクトリのunitディレクトリ以下のPHPUnit_Test_Caseを自動実行します。 テストクラスファイルの更新を検知しテストを実行します。また、テストクラスファイルに対応する対象クラスファイルの更新を検知しテストを自動実行します。


s2sqlite

コマンドリファレンス

% symfony s2qlite [環境指定]
  • 環境指定
    symfonyの環境を指定します。この値に従ってプロジェクトディレクトリ/config/ディレクトリのpdo.diconが使用されます。 デフォルト値は [prod] です。

説明

　環境指定で指定されたpdo.diconのdsnがSqliteの場合に、Sqliteデータベースへの簡易コンソールを起動します。
